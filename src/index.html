Units

Prefer these units
  • %
  • em
  • px
Avoid these units
  • in
  • cm
  • mm

Color

Don't use color keywords, instead use hexadecimal color codes.

Don't use shorthand hexadecimal color codes.

Do write hexadecimal color codes lowercase.

Bad 
                        
color: red;
color: #0f0;
color: #0000FF;
Good 
                        
color: #ff0000;
color: #00ff00;
color: #0000ff;

Border

Don't use border: none; or border: zero;. They are both valid options, but border: 0; is shorter.

Bad 
                        
border-width: none;
or 
                        
border-width: zero;
Good 
                        
border-width: 0;

Shorthand Attributes

Do use shorthand attributes in cases like margin, padding or border-width.

Bad 
                        
margin: 4px 4px 4px 4px;
padding: 20px 10px 20px;
Good 
                        
margin: 4px;
padding: 20px 10px;

But try to avoid shorthand attributes in cases like border or flex.

Bad 
                        
border: 5px solid #ff0000;
Good 
                        
border-width: 5px;
border-style: solid;
border-color: #ff0000;

Leading zero

Exclude unnecessary leading zeros.

Bad 
                        
color: rgba(24, 12, 76, 0.4);
Good 
                        
color: rgba(24, 12, 76, .4);

Unnecessary Significand

Don't add zeros to number values that are not fractions.

Bad 
                        
transition-duration: 1.0s;
Good 
                        
transition-duration: 1s;

Space after commas

Commas in lists must be followed by a space.

Bad 
                        
color: rgb(255,255,255);
Good 
                        
color: rgb(255, 255, 255);

Zero Units

Don't use units on 0 values.

Bad 
                        
margin: 0px;
Good 
                        
margin: 0;

Each partial contains the object defintition along with any of the associated elements and modifiers.

Pre-Processor

Use gulp.js as task runner.

Plugins

  • gulp-sass
  • gulp-sourcemaps

gulp-sass is a very light-weight wrapper around node-sass, which in turn is a Node binding for libsass, which in turn is a port of Sass.

Basic Setup with Source Maps and BrowserSync 
						
'use strict';

const gulp = require('gulp');
const sass = require('gulp-sass');
const sourcemaps = require('gulp-sourcemaps');

gulp.task('sass', function () {
	return gulp.src('./sass/**/*.scss')
		.pipe(sourcemaps.init())
		.pipe(sass().on('error', sass.logError))
		.pipe(sourcemaps.write('./maps'))
		.pipe(gulp.dest('./css'));
});

gulp.task('sass:watch', function () {
	gulp.watch('./sass/**/*.scss', ['sass']);
});

Class Naming Conventions

Kebab Case

All words lowercase with - separating words.

Example 
                        
.this-is-a-selector {

}

BEM

B(lock)__E(element)_M(odifier)

Don't use CSS tag or ID selectors when using BEM.

Block

  • A functionally independent page component that can be reused. In HTML, blocks are represented by the class attribute.
  • The block shouldn't influence its environment, meaning you shouldn't set the external geometry (margin) or positioning for the block.
  • Blocks can be nested in each other.
Example 
                        
<form class="search-form"></form>

Element

  • A composite part of a block that can't be used separately from it.
  • The element name is separated from the block name with a double underscore (__).
  • Elements can be nested inside each other.
  • An element is always part of a block, not another element. This means that element names can't define a hierarchy such as block__elem1__elem2.
Example 
                        
<form class="search-form">
	<input class="search-form__input"> 
	<button class="search-form__button">Search</button>
</form>

Modifier

  • An entity that defines the appearance, state, or behavior of a block or element.
  • The modifier name is separated from the block or element name by a single underscore (_).
  • Used when only the presence or absence of the modifier is important, and its value is irrelevant. For example, disabled. If a Boolean modifier is present, its value is assumed to be true.
Example 
                        
<form class="search-form search-form_focused">
	<input class="search-form__input">
	<button class="search-form__button search-form__button_disabled">Search</button>
</form>
